# Support

## Getting Help

Use GitHub Issues for bugs and feature requests.

For faster triage, include:

- What you were trying to do
- Exact command used
- Relevant config snippets (without secrets)
- Full error output
- OS and Python version

## Before Opening An Issue

1. Read the README quickstart.
2. Confirm dependencies are installed in a clean virtual environment.
3. Re-run with verbose logs and copy complete output.
4. Check if a similar issue already exists.

## Response Expectations

Maintainers aim to provide an initial response within 72 hours for new issues.
